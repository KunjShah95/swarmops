<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| validated diff → GitHub mutation | The system writes to an external repository here. |
| run API → orchestrator | A user-triggered request can start a high-impact automation path. |
| PR body → reviewers | Generated text may leak sensitive context if not filtered. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-04 | Tampering | PR opener | mitigate | Require a validated diff, branch name allowlist, and explicit repository scope before committing. |
| T-02-05 | Information Disclosure | PR body generation | mitigate | Redact tokens, env names, and secrets from summaries before opening the PR. |
| T-02-06 | Spoofing / Elevation of Privilege | run API | mitigate | Require repo-scoped credentials and verify the requested repository matches the configured target. |
</threat_model>
