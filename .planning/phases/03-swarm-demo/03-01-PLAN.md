<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| backend event stream → browser | Untrusted run data enters the UI here. |
| agent text → DOM | Generated text could become an XSS vector if rendered unsafely. |
| dashboard session → run data | One viewer should not see another viewer’s live run by accident. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-03-01 | Information Disclosure | stream route | mitigate | Emit only normalized events and avoid exposing raw prompt history, tokens, or secret headers. |
| T-03-02 | Tampering / XSS | agent timeline and diff viewer | mitigate | Escape all streamed text, render code/diffs through safe components, and never inject raw HTML. |
| T-03-03 | Spoofing | run session handling | mitigate | Scope stream subscriptions to the active run id and require authentication for live session access. |
</threat_model>
