<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| GitHub issue payload → backend | Untrusted text, labels, and metadata cross into the system here. |
| backend → Azure OpenAI / Azure AI Search | Prompt material and repo snippets leave the trusted boundary. |
| backend logs → operator | Sensitive issue text or token fragments could accidentally be written to logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-01 | Tampering / Spoofing | issue ingestion | mitigate | Validate payloads with typed models, reject missing repo identifiers, and never trust GitHub bodies without normalization. |
| T-01-02 | Information Disclosure | config and adapters | mitigate | Keep secrets in env vars, avoid logging raw issue bodies or auth headers, and redact client debug output. |
| T-01-03 | Elevation of Privilege | health and adapter surface | accept | A health endpoint is intentionally public; it exposes no privileged state and carries no write path. |
</threat_model>
