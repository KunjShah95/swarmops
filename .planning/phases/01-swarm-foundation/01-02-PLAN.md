## Trust Boundaries

| Boundary | Description |
|---|---|
| issue/context payload → agents | Untrusted content becomes prompt material here. |
| code writer output → execution guard | Generated patches cross into a validation boundary here. |
| orchestrator state → later streaming | Intermediate results could leak if not normalized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-04 | Tampering | planner prompt | mitigate | Require file:line evidence and structured fields so the model cannot invent paths without failing validation. |
| T-01-05 | Elevation of Privilege | code execution guard | mitigate | Use a minimal sandboxed validation path, avoid shell expansion, and reject anything that requires arbitrary command execution. |
| T-01-06 | Information Disclosure | orchestrator state | mitigate | Emit only normalized turn records and keep raw prompt material out of the persisted state stream. |
</threat_model>
